# モジュールドキュメントルール

すべてのモジュールは、役割・責任・インタフェースを明記したドキュメントを必ず持つ。

## 対象

- コアモジュール（`crates/` 配下）
- 汎用モジュール（`Ars-Module` 各ブランチ）
- プラットフォームブリッジ（`Ars-PlatformPlugin`）
- プラグイン（`ars-plugin-*`）

## 必須ドキュメント

各モジュールのルートに `README.md` を配置し、以下のセクションを含めること。

### 1. 概要（Overview）

- モジュールの目的を 1〜2 文で記述
- このモジュールが **何をするか** と **何をしないか**

### 2. ドメイン（Domain）

- このモジュールが属するドメイン領域
- 扱う概念・用語の定義（ユビキタス言語）

### 3. 責務（Responsibilities）

- このモジュールが担う責務を箇条書きで列挙
- 責務の境界を明確にする（「〜は本モジュールの責務ではない」）

### 4. 公開インタフェース（Public Interface）

- 公開する trait / struct / 関数の一覧
- 主要な型のシグネチャと用途の簡潔な説明

### 5. 依存関係（Dependencies）

- 依存先モジュール・クレートの一覧
- 依存の方向と理由

### 6. 使用例（Usage）

- `Cargo.toml` の依存記述
- 最小限の利用コード例

## テンプレート

```markdown
# <モジュール名>

## 概要

<1〜2文でモジュールの目的を記述>

## ドメイン

- **領域**: <ドメイン名>
- **用語**:
  - `<Term>`: <定義>

## 責務

- <責務1>
- <責務2>
- ~~<責務外>~~（本モジュールの責務ではない）

## 公開インタフェース

| 種別 | 名前 | 用途 |
|------|------|------|
| trait | `XxxRepository` | <用途> |
| struct | `Xxx` | <用途> |
| fn | `do_something()` | <用途> |

## 依存関係

| 依存先 | 理由 |
|--------|------|
| `ars-core` | ドメインモデル参照 |

## 使用例

\`\`\`toml
[dependencies]
<module> = { ... }
\`\`\`

\`\`\`rust
use <module>::*;
// 最小限の利用コード
\`\`\`
```

## 運用ルール

1. **モジュール新設時**: PR にドキュメントを含めること。ドキュメントのないモジュールはマージしない
2. **インタフェース変更時**: 公開インタフェースを変更した場合、ドキュメントも同時に更新する
3. **レビュー観点**: PR レビューではドキュメントの有無と正確性を確認する
