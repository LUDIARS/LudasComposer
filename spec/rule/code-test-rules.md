# コードテストルール

生成されたコードをビルド・テストし、エラーを修正するためのルール。

> **位置づけ**: コード生成フローの第4フェーズ（最終フェーズ）。第3フェーズで生成したコードの
> ビルド確認・テスト実行を行い、エラーがあれば修正する。

## テストフロー

```
[生成されたソースコード]
    ↓ ビルド
[コンパイル / 型チェック]
    ↓ エラーあり → 修正 → 再ビルド（成功するまで繰り返し）
    ↓ 成功
[テスト実行（テストが存在する場合）]
    ↓ 失敗あり → 修正 → 再テスト（成功するまで繰り返し）
    ↓ 全テスト成功
[結果報告]
```

## 入力

| ソース | 内容 |
|--------|------|
| 生成コードディレクトリ | 第3フェーズで出力されたソースコード |
| `codedesign/` | 設計ファイル（修正時の参照用） |
| `spec/platforms.md` | プラットフォーム定義（ビルドコマンドの決定） |
| 本ルール | テスト・修正の方針 |

## プラットフォーム別ビルドコマンド

| プラットフォーム | ビルド | テスト |
|---------------|-------|-------|
| Unity | Unity Editor のコンパイル確認、または `dotnet build` | `dotnet test` / Unity Test Runner |
| Godot | Godot Editor のパース確認 | `godot --headless --run-tests` |
| Unreal | `UnrealBuildTool` / `dotnet build` | `UnrealEditor -ExecCmds="Automation RunTests"` |
| Ergo | `npx tsc --noEmit` | `npx vitest run` |

## テスト実行ルール

### T1: まずビルドを通す

コード生成後、最初にビルド（コンパイル / 型チェック）を実行する。
ビルドが通らない場合、テストの実行には進まない。

### T2: ビルドエラーは自動修正する

ビルドエラーが発生した場合、以下の手順で修正する：

1. エラーメッセージを読み取り、原因を特定する
2. 該当ファイルを修正する
3. 再ビルドして確認する
4. 成功するまで繰り返す

修正時は codedesign の設計意図を維持すること。
設計の変更が必要な場合は `// TODO: codedesign要確認 - {理由}` コメントを残す。

### T3: テストがある場合は実行する

codedesign のテストケースから生成されたテスト、またはプロジェクトに既存のテストがある場合は実行する。
テストが存在しない場合はビルド成功をもって完了とする。

### T4: テスト失敗は自動修正する

テストが失敗した場合、以下の手順で修正する：

1. 失敗したテストのエラーメッセージを確認する
2. テストコードではなく **実装コード** を修正する（テストが正しい前提）
3. テストの期待値が codedesign と矛盾する場合のみテストコードを修正する
4. 再テストして確認する
5. 成功するまで繰り返す

### T5: 修正の上限

ビルドエラー・テスト失敗の修正は **最大5回** まで繰り返す。
5回で解決しない場合は、残存エラーを報告してユーザーに判断を委ねる。

## 結果報告

### ビルド・テスト成功時

```
## テスト結果

- **ビルド**: 成功
- **テスト**: {n} 件成功 / {n} 件中
- **修正回数**: {n} 回

### 修正履歴（修正があった場合）
| # | 種別 | ファイル | 内容 |
|---|------|--------|------|
| 1 | ビルドエラー | player.cs | 型不一致を修正 |
| 2 | テスト失敗 | health-system.cs | 初期値の誤りを修正 |
```

### 解決できないエラーがある場合

```
## テスト結果

- **ビルド**: {成功 / 失敗}
- **テスト**: {n} 件成功 / {m} 件失敗 / {total} 件中
- **修正回数**: 5 回（上限到達）

### 未解決エラー
| # | 種別 | ファイル | エラー内容 | 推定原因 |
|---|------|--------|-----------|---------|
| 1 | テスト失敗 | enemy-ai.cs | Assert.AreEqual failed | codedesign の仕様確認が必要 |

上記のエラーは自動修正できませんでした。確認をお願いします。
```
