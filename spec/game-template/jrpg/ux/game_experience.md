# jrpg — ゲーム体験 (全体設計)

## マクロな体験曲線

```
プレイ時間 →
  Hours 0-3:    操作覚える + 序盤戦闘 + 主人公動機提示
  Hours 3-15:   仲間集結 + 中盤クエスト + 装備更新サイクル
  Hours 15-30:  世界観拡大 + 中ボス連戦 + サブストーリー解禁
  Hours 30-50:  終盤フェーズ + クライマックス + 最終ダンジョン
  Hours 50-100: 裏ダンジョン / 周回 / やり込み / 隠しボス
  Hours 100+:   コミュニティ攻略 / 縛りプレイ / 二週目以降
```

各時期に **「いま何をする動機」** が明確に存在する設計が肝。

## 戦闘の頻度設計

| 区間 | エンカウント間隔 | 理由 |
|------|----------------|------|
| ダンジョン | 30-60s | 探索のリズム |
| 街道 / フィールド | 60-120s | 移動の主目的 |
| 街中 | なし | 物語 / イベント中心 |
| 安全エリア (休息地点) | なし | プレイヤーに息継ぎ |

シンボルエンカウントなら 「敵を見て避けるかぶつかるか」 の **能動性** がプラス。

## レベル / 報酬カーブ

```
Lv  必要 XP/up   主要 stat (例: 攻撃)
1     50          10
5     200         30
10    600         60
20    2000        120
30    5000        200
50    15000       400
70    35000       650
99    100000      900
```

- 序盤は **1-2 戦で Lv up**、 育成感を即体験
- 中盤は **5-10 戦で Lv up**、 装備サイクルとの兼ね合い
- 終盤は **30-50 戦で Lv up**、 やり込み層向け

## ボス vs 雑魚の設計

| | 雑魚 | ボス |
|---|----|-----|
| HP | 1-3 ターンで沈む | 8-15 ターンで沈む |
| 行動 | 1 種 / 単純 | 多段階パターン |
| BGM | 通常戦闘 | 専用 (フェーズ移行で再加速も) |
| 演出 | 通常 | 撃破時カットイン + 報酬演出 |
| 報酬 | XP + 通貨 + 通常ドロップ | XP 大 + レア装備 + 物語進行 |

## 配信 / 共有適性

- **長尺なのでセクション切り出し** が重要 → 各章ごとにセーブ可
- **ストーリーシーン** は SNS 切り抜き素材になる (台詞 + 表情)
- **レアアイテム** や **レア戦闘ドロップ** はリアクションが取れる
- **ジョブ / ビルドの自由度** を持たせると配信者ごとの個性が出せる

## アンチパターン

| アンチ | 理由 |
|--------|------|
| 「全イベント自動進行のみ」 | プレイヤー参加感ゼロ、 視聴ゲーム化 |
| 「弱点ヒット・ヒットの嵐」 | 戦闘の戦略性消失、 殴るだけゲーム |
| 「セーブが街でしかできない」 | ダンジョン途中の事故時にロス大 |
| 「メインクエストのみで横道なし」 | ジャンルの世界観構築の機会喪失 |
| 「クリア後コンテンツ皆無」 | 達成感のピークから落差で離脱 |
