# jrpg — 個別 Feature 詳細 (UX 寄与)

## command-battle

- **狙う UX**: 「次に何をするか」 を **指揮官として選ぶ** 楽しさ
- **不調時の問題**:
  - ATB が速すぎる → 慌てて選ぶ → 戦術崩壊
  - ターン制で待ち時間長すぎ → だれる
- **チューニング**: ATB 1 ゲージ満タン 4-6s が定石、 重要選択肢は静止

## stat-system

- **狙う UX**: HP/MP/攻/防 が **数値で見える** ことの納得感
- **不調時の問題**:
  - パラメータ多すぎ → プレイヤーが理解放棄
  - 上昇幅が小さすぎ → 成長を感じない
- **チューニング**: 主要 6 項目以内、 Lv up で +5-10% 増

## leveling

- **狙う UX**: 「**苦戦 → 撃破 → レベルアップ → 次** 」 のループによる達成感
- **不調時の問題**:
  - 必要 XP が大きすぎ → 「育てゲー疲労」
  - 小さすぎ → 戦闘の重みがなくなる
- **チューニング**: Lv 1-10 は 5 戦/up、 Lv 50-99 は 30-50 戦/up

## dialogue-system

- **狙う UX**: キャラの個性がセリフで **生きている** 印象
- **不調時の問題**:
  - 自動再生速度がプレイヤーに合わない → 飛ばす / 待たされる
  - 選択肢の効果が見えない → 選ぶ意味喪失
- **チューニング**: 表示速度 30 文字/秒 + ユーザー設定で 1.5-3x 可

## quest-system

- **狙う UX**: 常に **次の目的が指南** されていて迷わない
- **不調時の問題**:
  - クエスト目的不明 → 迷子
  - 目的地距離が遠すぎ → 単調な移動
- **チューニング**: 目的地までミニマップ表示、 移動 30s-3m 程度

## status-effect

- **狙う UX**: バフ / デバフが **戦術選択** を増やす
- **不調時の問題**:
  - 効果不明瞭 → 使われない
  - 強すぎ → 暴力的な勝ち方一択
- **チューニング**: 全状態異常を **3 ターン以内** で消える設計、 一発逆転は避ける

## elemental-affinity

- **狙う UX**: 弱点を見つけたとき 「やった分かった」 達成感
- **不調時の問題**:
  - 弱点不明 → 試行錯誤の苦痛
  - 弱点表示しすぎ → 緊張感ゼロ
- **チューニング**: 1 度ヒットすると以降 UI 表示、 初見は試す

## save-load

- **狙う UX**: 任意セーブ + 自動セーブで **不安なく長時間遊べる**
- **不調時の問題**:
  - 自動セーブのみ → ダンジョンで間違えた時にやり直せない
  - 任意セーブのみ → セーブ忘れで進捗ロス
- **チューニング**: 街 / ダンジョン入口 / ボス前で auto、 任意セーブは常時 (戦闘中除く)

## crafting (任意)

- **狙う UX**: 素材集めの **副次的目標** + 装備の自由度
- **不調時の問題**:
  - 素材集めが過酷 → ロード作業化
  - クラフト効率がショップを超える → 経済破綻
- **チューニング**: 序盤は売買有利、 中盤以降クラフト価値、 終盤独占
