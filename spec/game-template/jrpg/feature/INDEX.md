# jrpg / feature — 機能リスト

このジャンル (jrpg = ストーリー型 JRPG) で必要となる機能と UX 寄与を列挙。

| Feature ID | 必須/任意 | UX 寄与 |
|------------|---------|---------|
| `command-battle` | 必須 | **じっくり考える** 戦闘 — ATB / ターンの選択時間がプレイヤーに 「自分が指揮している」 感覚を与える |
| `stat-system` | 必須 | レベル / 装備の差が **数値で見える** ことの満足感。 成長を実感する根本 |
| `party-system` | 必須 | キャラ間の **役割分担** とパーティ編成の自由度。 「自分の戦い方」 を作る |
| `leveling` | 必須 | 経験値で**着実に強くなる** 報酬ループ。 探索 + 戦闘の動機 |
| `skill-learning` | 必須 | レベル / SP で **新しい選択肢が増える** という具体的な成長報酬 |
| `equipment-slots` | 必須 | 装備差し替えで **パラメータ + 見た目** が変わる楽しさ |
| `dialogue-system` | 必須 | キャラ会話で **物語に入り込む** 体験の核。 選択肢で参加感 |
| `event-script` | 必須 | カットシーン演出で **重要シーンの高揚** を作る |
| `quest-system` | 必須 | メイン / サイドクエストで **次の目的が常にある** 状態を維持 |
| `inventory` | 必須 | アイテム管理 — 戦利品の **物質感** で達成感を増幅 |
| `shop` | 必須 | 通貨で **明確な選択** (買うか節約か) を迫る |
| `status-effect` | 必須 | 状態異常で戦闘に **戦術の幅** を出す |
| `elemental-affinity` | 必須 | 弱点 / 耐性で **敵分析の楽しみ** を作る |
| `encounter` | 必須 | 探索中の戦闘発生 — **緊張と弛緩のリズム** |
| `overworld` | 必須 | 街 / ダンジョン / 世界マップで **物語の地理感覚** |
| `crafting` | 任意 | 素材 → 装備の **二次的成長軸** |
| `save-load` | 必須 | **任意セーブ + オートセーブ** の安心感。 物語を 「自分のペースで」 進める |
| `level-up-screen` | 必須 | レベルアップ時の **明確な祝福演出** |
