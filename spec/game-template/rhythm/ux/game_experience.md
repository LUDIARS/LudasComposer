# rhythm — ゲーム体験

## セッション設計

- 1 曲 1.5-3 分 → セッションは 5-30 曲
- 体力 (BP) や気力 (Stamina) で **疲れて止める** メカニクスを入れるか入れないか:
  - 入れる (アーケード式): プレイ時間管理が自動。 課金回復文化
  - 入れない (家庭機 / PC): 自由に何時間でも

## 譜面の難度設計

| 難度 | BPM | ノート密度 | 種類 | 想定プレイヤー |
|------|-----|----------|------|----------------|
| Easy | 80-130 | 60-100 NPS=1 | Tap, Hold | 初心者 |
| Normal | 110-150 | 100-180 NPS=2 | + Slide | カジュアル |
| Hard | 140-180 | 180-300 NPS=3-5 | + Flick / 複雑配置 | 中上級 |
| Expert | 160-220 | 300-450 NPS=6-9 | 全種 / 二重 | 上級 |
| Master | 180-260 | 400-600 NPS=8-12 | 高密度連打含む | エキスパート |

NPS = Notes Per Second。 体感上 8 を超えると 「常人不可」 領域。

## 楽曲ライブラリ設計

- 入門: 30 曲 (発売時)
- 月次追加: 4-8 曲
- ジャンル分散: J-Pop / EDM / Anime / Game OST / Vocaloid / Original
- BPM 分散: 80-260 を満遍なく

## 配信 / 共有適性

- フルコンボ達成画面が **SNS 切り抜き** に最適
- 高難度 Expert クリアは観戦者リアクション大
- スコアアタック / ランキングを **観戦** できると盛り上がる
- 譜面エディタ + コミュニティで **長期生存**

## メタ進行 (任意)

- 楽曲アンロック: 別曲を S ランクで解禁、 等
- コスチューム / アバター: スコア達成で解禁
- ストーリーモード: 曲を進めると物語が進む (ボカコレ系)
- 称号: 達成バッジ

## アンチパターン

| アンチ | 理由 |
|--------|------|
| 「キャリブレーションなし」 | 環境次第で完全に遊べない人が出る |
| 「判定窓を体感より広く」 | 「PERFECT 連発で達成感ゼロ」 |
| 「ランキング降格システム」 | ライト層が脱落 |
| 「課金で判定が緩くなる」 | 信用喪失 |
| 「コミュニティ作譜不可」 | コンテンツ寿命の天井が早い |
