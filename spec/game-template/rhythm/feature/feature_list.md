# rhythm — 個別 Feature 詳細 (UX 寄与)

## audio-engine + audio-sync

- **狙う UX**: 音とノーツの **一体感**。 「叩いた瞬間音が鳴る」 体験
- **不調時の問題**:
  - 出力レイテンシ大 → 「叩いてから音が遅れる」 → 致命的に冷める
  - キャリブレーション欠如 → 環境ごとのズレで PERFECT 取れず → やる気喪失
- **チューニング**: ASIO / WASAPI / Core Audio で 10ms 未満を目標、 起動時に 「タップ × 8」 で自動キャリブ + 設定で手動微調整 (-200..+200ms)

## timing-judge

- **狙う UX**: 「合った」 / 「ズレた」 を **見て体感** できる
- **不調時の問題**:
  - 窓が広すぎ → 適当でも PERFECT → 上達感喪失
  - 窓が狭すぎ → 達者でも GOOD 連発 → 諦観
- **チューニング目安**: PERFECT ±25ms / GREAT ±60ms / GOOD ±120ms (高難度では PERFECT を ±15ms に)

## combo-counter + score

- **狙う UX**: コンボが伸びるたび **興奮の高まり**
- **不調時の問題**:
  - フルコンボ条件が曖昧 → 達成感が湧かない
  - スコア倍率が単純すぎ → 上手いプレイの差が出ない
- **チューニング**: コンボ N 段階で UI 派手に / SE 高音、 100 / 200 / 500 でステップアップ

## note-types (Tap / Hold / Slide / Flick)

- **狙う UX**: 譜面に **多様性** と **音楽表現の対応**
- **不調時の問題**:
  - Tap だけ → 譜面が単調
  - 種類多すぎ → 初心者がついていけない
- **チューニング**: 入門譜面は Tap + Hold のみ、 高難度で Slide / Flick 解禁

## note-renderer

- **狙う UX**: ノートが **正確な位置 / 速度** で来る視覚精度
- **不調時の問題**:
  - フレーム落ちでジッタ → 判定不能
  - 視覚と音のズレ → user_offset で補正できる範囲を超える
- **チューニング**: 60 fps を絶対死守、 120 fps が望ましい。 描画は audio clock 起点で時刻ベース

## song-select-ui

- **狙う UX**: 「次に何をやるか」 を **選ぶ楽しさ**。 プレビュー試聴で気分を作る
- **不調時の問題**:
  - 曲数が多くてフィルタが弱い → 探せない
  - プレビュー再生のローディング遅い → ストレス
- **チューニング**: ジャンル / 難易度 / レベル別タブ + プレビュー 5s 以内に開始

## chart-editor (任意)

- **狙う UX**: コミュニティ作譜で **コンテンツ無限化**
- **不調時の問題**:
  - 操作難解 → 投稿数伸びない
  - レビュー機能なし → 質が散る
- **チューニング**: チュートリアル譜面 / 採点 / コミュニティ評価 / 投稿フローを最初から想定
