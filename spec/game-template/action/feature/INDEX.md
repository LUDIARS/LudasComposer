# action / feature — 機能リスト

このジャンル (action) で必要となる機能を、 **何を** 実現するか + **UX へどう寄与する** か とともに列挙する。 詳細仕様は [`spec/game-lexicon/features/`](../../../game-lexicon/) を参照。

## サマリ

| Feature ID | 必須/任意 | UX 寄与 |
|------------|---------|---------|
| `input-buffer` | 必須 | 「入れたい技がブッキングしないでつながる」 という気持ち良さ。 60 fps のシビアな入力差を吸収して **コンボ成功体験** を支える |
| `hitbox-system` | 必須 | 命中 / 被弾の **正当性感覚**。 「明らかに当たってないのに死んだ」 を防ぐ |
| `health-system` | 必須 | プレイヤー / 敵の HP の見える化と **死亡** という強い結末。 緊張感の核 |
| `combo-system` | 必須 | キャンセル可能フレームを使いこなす **習熟感**。 「自分が上達している」 を実感させる |
| `dodge-roll` | 必須 | i-frame による **緊急回避** = ピンチからの逆転。 達成感と緩急 |
| `hitstop` | 必須 | 命中の瞬間の **一瞬の停止** で打撃感を増幅。 「重い一発」 を見せる |
| `camera-follow` | 必須 | プレイヤー視点を快適に保つ。 **戦闘中も状況把握できる** UX の土台 |
| `lockon` | 推奨 | 近接敵を選んで集中できる。 操作量を減らし **戦術選択に集中** させる |
| `parry-system` | 推奨 | 紙一重防御による **大きな見返り** (敵スタガー)。 リスク・リワード設計の起点 |
| `posture-stagger` | 推奨 | 体幹ゲージで **削り合いに別軸** を導入。 SEKIRO 系の独自体験 |
| `enemy-ai-states` | 必須 | 敵に「いま何をしているか」 を **読みやすく** する。 戦術立案の前提 |
| `stamina-system` | 任意 | 行動に **コスト** を課して 「常に最強行動」 を防ぐ。 緩急設計の道具 |
| `save-load` | 必須 | 区切り (篝火 / チェックポイント) で再挑戦できる **挑戦性のループ** を実現 |
