# action — 個別 Feature 詳細 (UX 寄与)

各 Feature が **狙う UX** と **実装が外れたとき / 数値が悪いとき** に起こる UX 問題を併記する。
チューニング指針として使う。

## input-buffer

- **狙う UX**: 「入れたい技を入力したのに出ない」 を回避する
- **不調時の UX 問題**:
  - 窓が短すぎる → 「入れたのに出ない」 苛立ち
  - 窓が長すぎる → 「入れた覚えのない技が出る」 暴発
- **チューニング目安**: 100-200ms (60 fps で 6-12 frames)

## hitbox-system

- **狙う UX**: ヒット / ミスを **見た目通り** に解決する
- **不調時の UX 問題**:
  - 当たってないのに食らう → 「理不尽だ」
  - 明らかに当たってるのに当たらない → 「狙ってもムダ」
- **チューニング目安**: 攻撃 hitbox は見た目より **わずかに大きく**、 食らい (hurtbox) は見た目より **わずかに小さく** が定石

## health-system

- **狙う UX**: 生死の境界を緊張感ある形でフィードバック
- **不調時の UX 問題**:
  - HP 表示の更新が遅い → 「いま死んだのか分からない」
  - 死亡時の演出が弱い → 死の重さが伝わらない
- **メモ**: 残 HP 20% 以下で BGM / 心拍 SE / 画面赤化など 「ピンチ演出」 を用意するのが定石

## combo-system

- **狙う UX**: 上達したプレイヤーが「自分のテクニックで時短 / 高威力」 を体験する
- **不調時の UX 問題**:
  - キャンセル窓が短すぎる → 「入力が完璧でも繋がらない」
  - キャンセル窓が長すぎる → 「ボタン連打でも繋がる」 → 上達感喪失
- **チューニング目安**: クラスごとに技別の cancel_window を定義、 上級技ほど短めに

## dodge-roll

- **狙う UX**: 死にそうな状況からの **逆転** を可能にする
- **不調時の UX 問題**:
  - i-frame が短すぎる → 「ローリングしても食らう」 → 諦観
  - i-frame が長すぎる → 「ローリングしてれば勝てる」 → 戦術崩壊
- **チューニング目安**: 300-500ms の i-frame + 600ms 程度のクールダウン + スタミナ消費

## hitstop

- **狙う UX**: 「いま当たった」 という**インパクト** を増幅
- **不調時の UX 問題**:
  - 短すぎ → 当たった感が薄い (爽快感ない)
  - 長すぎ → テンポが切れる、 戦闘がもたつく
- **チューニング目安**: 50-150ms。 ダメージや技の重さで可変

## camera-follow

- **狙う UX**: 戦闘中もプレイヤーが盤面を把握できる
- **不調時の UX 問題**:
  - 速すぎる追従 → 酔う
  - 遅すぎる追従 → 自キャラが画面から消える
  - シェイクが強すぎる → 視認性悪化
- **チューニング目安**: smoothing 0.1-0.2、 シェイクは振幅 0.05-0.2m / 100-200ms

## lockon

- **狙う UX**: 「敵を選ぶ」 操作負担を下げて **戦術** に集中させる
- **不調時の UX 問題**:
  - ロック切替が硬直する → 多敵戦で混乱
  - ロック対象が画面外に行く → 「何を見てるの?」
- **チューニング目安**: ロック中は緩い旋回 + 距離超で解除、 切替は右スティック / R3

## parry-system

- **狙う UX**: 紙一重の **見切り成功** に大きな見返り (敵スタガー / 隙)
- **不調時の UX 問題**:
  - 受付窓が広すぎる → 連打パリィで全部止まる
  - 狭すぎる → 偶然以外成功しない → 採用されない
- **チューニング目安**: 攻撃の startup 直前 + 8-12 frames

## posture-stagger

- **狙う UX**: HP 削りとは別の **「敵を崩す」 リソース** を導入し、 攻防の駆け引きを増やす
- **不調時の UX 問題**:
  - 回復が早すぎる → 蓄積する意味なし
  - 0 化のリターンが弱い → そもそも狙わない
- **チューニング目安**: 0 化で 「忍殺チャンス」 等の 1 撃必殺、 回復は秒 5-10%

## enemy-ai-states

- **狙う UX**: 敵が「読める」 → プレイヤーは戦術を立てる → 上達感
- **不調時の UX 問題**:
  - 状態遷移が見えない → 「敵が何やるかわからない」
  - 状態が単調 → 飽きる
- **チューニング目安**: 各 state に**わかりやすい予兆** (姿勢 / SE / VFX) を付ける

## stamina-system (任意)

- **狙う UX**: 「常に強行動」 を防ぎ、 **緩急のリズム** を作る
- **不調時の UX 問題**:
  - 消費が大きすぎる → 「何もできない」 ストレス
  - 消費が小さすぎる → リソース管理が形骸化
- **チューニング目安**: 通常攻撃 5-10 / 重攻撃 15-25 / 回避 20-25 / 回復 1 秒 25-40

## save-load (篝火型)

- **狙う UX**: 「死んでも進める」 安心感 + 「ここまで戻される」 緊張感の両立
- **不調時の UX 問題**:
  - 篝火間隔が長い → 死亡時のロスが大きすぎてやる気喪失
  - 篝火間隔が短い → 緊張感が薄い
- **チューニング目安**: 通常 3-5 分の道のりごとに 1 つ、 ボス前に必ず 1 つ
