# action — ユビキタス言語

| 用語 | 説明 |
|------|------|
| アクション (Action) | プレイヤー / 敵が遷移可能な 1 つの行動単位。 攻撃 1 段、 回避 1 回など |
| キャンセル | 進行中のアクションを途中で別アクションに切り替えること。 通常は **キャンセル可能フレーム** が定義された区間でのみ可 |
| キャンセル窓 (Cancel Window) | キャンセルが受け付けられるフレーム範囲 |
| ヒットボックス (Hitbox) | 攻撃側が 「当たり」 を発生させる形状 |
| ハートボックス (Hurtbox) | 受け側が 「ダメージを受ける」 形状 |
| プッシュボックス (Pushbox) | キャラ同士が重ならないよう押し戻すコリジョン |
| i-frame (インビンシブルフレーム) | この期間中はダメージを受けない無敵時間 |
| ヒットストップ (Hitstop) | 命中時に時間を止める演出。 通常 1-10 frames |
| フレームデータ | 技ごとの startup / active / recovery 等の数値表 |
| startup (発生) | 入力から攻撃判定が出るまでのフレーム数 |
| active (持続) | 攻撃判定が出ている期間 |
| recovery (硬直) | 判定終了から次行動可能までのフレーム数 |
| 体幹 / Posture | スタガー (崩し) を引き起こす蓄積ゲージ |
| スタガー / 崩し | 体幹が満タンで発生する一時的な行動不能状態 |
| パリィ (Parry) | 攻撃の発生直前にガードして敵をスタガーさせる成功的防御 |
| ロックオン (Lock-on) | 視点と入力方向を特定の敵に向ける補助システム |
| 篝火 (Bonfire) | チェックポイント兼セーブ地点 (ダクソ由来呼称) |
| 忍殺 (Shinobu / Execution) | スタガー / 弱点状態の敵に対する一撃必殺技 |
| 起き攻め | ダウン中の相手の起き上がりに合わせて重ねる攻撃 |
| 確反 (確定反撃) | 相手の硬直が大きく確実に反撃が間に合う状況 |
| ダウン値 | 連続攻撃で蓄積し、 一定値で強制ダウンする内部値 (一部タイトル) |
| カメラシェイク | ヒット / 爆発時に視点を揺らす演出 |

## 命名ルール

- コードでは **英語スネークケース**: `cancel_window_ms`, `hit_stop_frames`
- データ (TOML / JSON) でも同様
- 仕様書 / プランナー資料では **日本語 + 英語併記** を推奨
- 用語が増えた場合は本ファイルにのみ追記し、 他資料は ID で参照する
