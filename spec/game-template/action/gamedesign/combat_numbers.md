# 戦闘数値テーブル指針

## プレイヤー基準値 (Lv1)

| 項目 | 値 | 備考 |
|------|----|------|
| 最大 HP | 100 | UI 表示は 100 = 1 セグ |
| 最大スタミナ | 100 | regen: 25/s (戦闘外) / 12/s (戦闘中) |
| 体幹最大 | 100 | regen: 5/s (静止時) |
| 通常攻撃ダメージ | 12 | Light: 12, Heavy: 28 |
| 通常攻撃 startup | 8 frames (Light) / 14 (Heavy) | 60 fps 換算 |
| 通常攻撃 active | 4 frames | |
| 通常攻撃 recovery | 12 frames (Light) / 22 (Heavy) | |
| 回避 i-frame | 24 frames (= 400ms) | |
| 回避 cooldown | 36 frames (= 600ms) | |
| 回避スタミナ | 25 | |

## 敵 (雑魚) 基準値

| 項目 | 値 |
|------|----|
| 最大 HP | 60 |
| 体幹最大 | 50 |
| 攻撃ダメージ | 8 |
| 攻撃発生 | 24 frames (予兆 18f + active 6f) |
| 攻撃硬直 | 30 frames (= 反撃確定窓 14f) |

## ボス基準値

| 項目 | 値 |
|------|----|
| 最大 HP | 800 (Phase1) + 700 (Phase2) |
| 体幹最大 | 200 |
| 攻撃ダメージ | 30-50 |
| パリィ可能技 | 70% (3/4 程度) |
| 体幹 0 で発生する処刑可能時間 | 90 frames |

## 数値設計の原則

1. **プレイヤー HP 100, 雑魚攻撃 8** → 12 発食らうと死亡 (実際は回復もあるので長い)
2. **ボス HP 1500, プレイヤー Light 12** → 切る回数で約 130 発 → 多すぎ → スタガー処刑で大ダメージ前提
3. **回避 i-frame 24f / 雑魚攻撃 active 6f** → タイミング合えば確実に避けられる
4. **回避 cooldown 36f / 敵連撃間隔 30f** → 連撃を 1 回しか避けられない → ガード or パリィを混ぜる動機

## バランス調整スプレッドシート

実装はスプレッドシート ↔ TOML を同期する仕組みを前提:

```
balance/
  ├─ player.toml      ベース値
  ├─ enemies/*.toml   敵種別ごと
  ├─ bosses/*.toml    ボスごと
  └─ scaling.toml     難易度別倍率
```

## 数値変更時のテストルーチン

`spec/game-template/action/test/` 配下:

- 自動化: 「Lv1 プレイヤー vs 雑魚 1 体、 何回攻撃で死ぬか」 を回帰テスト
- 自動化: 「ボス Phase1 が `time-to-kill` 範囲内か」 (60-180s)
- 手動: フィーリングチェック (ヒットストップが体感できるか)
