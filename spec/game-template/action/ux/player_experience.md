# action — プレイヤー体験

「**プレイヤーが何を感じるか**」 を時間軸に沿って書き出す。 個々の数値や演出は [`feature/`](../feature/INDEX.md) の各 Feature が担う。

## 戦闘 1 ターンの体験フロー

```
敵接近を察知 (0.5-1s 前から VFX/SE で予兆)
    ↓
プレイヤー: 「どう対応するか」 を選ぶ
    ├─ 攻撃 (リスクテイク) → ヒットストップ → 達成感
    ├─ 回避 (i-frame) → 逆転スキ → 次の攻め
    └─ ガード / パリィ → 微小スキ → スタガー (大成功)
    ↓
敵の状態遷移を観察し、 次行動を読む
    ↓
コンボ完走 / 敵スタガーから 「忍殺」 級の決め技
    ↓
キル成立 → 短い静止 + 強 SE/VFX で 「やった」 感
```

各分岐に対し:
- **失敗 (被弾)** すれば HP 削れ → ピンチ演出 (赤画面 / 心拍 SE)
- **成功** すれば敵が崩れる → 次の機会
を交互に積んで、 **緊張と達成の波** を作る。

## ボス戦の長期体験フロー

```
1 戦目: 死ぬ
   → 敵パターンを覚える
2-3 戦目: 数発当てて死ぬ
   → 「あと少し」 で続ける
4-5 戦目: フェーズ移行を見る
   → 「全パターン把握」
6-N 戦目: パターン理解 → 選択肢に集中
   → 撃破
クリア: 強い達成感 + 攻略動画を見たくなる SNS 反応
```

「**何度も死ねるが、 死ぬほど学べる**」 構造が核。 死亡時のロスは **学習量** に比例して **減る**。

## 「いま自分は上達している」 を感じさせる仕掛け

1. 入力 → 反応の **遅延 < 100ms** を死守 (UX が崩れる閾値)
2. ヒットストップ + シェイク + SE で **個々のヒットを記憶に残す**
3. ステージごとに **新しい技 / 仕掛け** を導入 (覚えるべきものを刻む)
4. ボス戦は **複数フェーズ** + **明確な行動予告** (覚えれば勝てる)
5. **死亡画面で原因を簡潔に表示** (「○○の攻撃でやられた」)

## NG パターン (避けるべき UX)

| NG | 起きる原因 | 対策 |
|----|-----------|------|
| 「理不尽だ」 | hitbox vs 見た目の不一致 / 敵 AI が読みにくい | feature/feature_list.md の調整 |
| 「飽きる」 | 戦術選択肢が少ない / 敵パターン単調 | parry / lockon / stagger の有無で抑揚 |
| 「酔う」 | カメラ追従 / シェイクが強すぎ | smoothing 上げる / シェイク振幅下げる |
| 「テンポが悪い」 | hitstop 長すぎ / アニメ硬直長すぎ | コアフレーム数で再見直し |
| 「死亡時に学べない」 | 原因表示なし / 篝火が遠い | UI で死因表示、 篝火間隔を縮める |
