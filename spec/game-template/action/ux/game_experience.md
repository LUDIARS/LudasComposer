# action — ゲーム体験 (全体設計)

「**ゲーム側が組み立てる体験の波**」 をマクロに書く。 individual な戦闘 1 回ではなく、 数時間-数十時間のセッションで何を狙うか。

## 体験の波 (難度曲線 + 報酬曲線)

```
難度 ───────────────────────────────────────────╮
                                              ╱│
                            ╱╲             ╱  │ ボス N (頂点)
              ╱╲           ╱  ╲           ╱
   ╱╲       ╱  ╲╲        ╱    ╲╲        ╱
__╱  ╲____╱     ╲╲______╱       ╲______╱
  C1   C2  C3 → ボス1   C4 → ボス2   C5

報酬 (新スキル / 新装備 / 新エリア)
  ↑       ↑                       ↑
  小      中                       大
```

- 難度は **山** で立ち上がり、 ボス前で頂点 → ボス撃破で **谷** (ご褒美エリア)
- ご褒美エリアでは新装備 / 新技解禁 → プレイヤーは 「これを試したい」 → 次の山へ
- 山の高さは **段階的に上がる** が、 ご褒美の **大きさも比例** して大きくする

## レベルデザインのリズム

ステージ単位で:

1. **イントロ** (新雑魚 1 種を 1-2 体)
   - 新行動を見せる + プレイヤーが安全に学習
2. **応用** (既習 + 新雑魚 + 高低差)
   - 学んだ対処を別状況で適用
3. **小休止** (篝火 / 取得アイテム)
   - 緊張をリセット
4. **混合戦** (複数種混合 + 数増)
   - 戦術を要求
5. **ミニボス**
   - 章ボスへの予習
6. **ボス**
   - 頂点

## プレイヤーモチベーション設計

時間軸ごとに何が動機になるか:

| 進行 | 主な動機 |
|------|---------|
| 開始-2h | 操作系統に慣れる + 序盤ボス撃破 |
| 2h-10h | スキル / 装備のビルド変更 + 中ボス連戦 |
| 10h-30h | 複数ビルドを試す + 探索 + やり込みボス |
| クリア後 | 高難度モード / NG+ / コミュニティ攻略共有 |

## 配信 / 共有適性

- ボス撃破時の **強い演出** (画面停止 + 文字 + SE) を必ず入れる → SNS 切り抜き素材
- 死亡画面で **原因の言語化** → 視聴者がツッコミやすい
- 攻略パターンに **発見の余地** を残す → 攻略動画の視聴動機
- ハイスピード操作の見栄え (DMC 系) or 静かな読み合い (ダクソ系) で映える方向を選ぶ

## アンチパターン

| アンチ | 理由 |
|--------|------|
| 「全行動可能、 全敵バフ」 | プレイヤーが何を学んでいいか分からない |
| 「死亡で 30 分巻き戻し」 | 諦観を生む。 学習モチベを損なう |
| 「ボスの行動完全ランダム」 | 学習が成立しない、 運ゲー化 |
| 「クリア後コンテンツ無し」 | 達成感のピークから空虚に落ちる |
| 「セーブが任意のみ」 | 死亡時の腹立ちが大きい (チェックポイント自動セーブを併設すべき) |
